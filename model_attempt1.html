Model 1

Find connectivity classification from distances

Define:

  • Cascade dam: Downstream & upstream dam within 15

  • FFR dam: No upstream, no downstream dam

  • Upstream dam: There is a downstream dam in 50km

  • Downstream dam: There is an upstream dam in 50km

# Connectivity classificantion column
dam_df <- dam_df %>% 
  mutate(connectivity_category = case_when(
    upstream_dam_dist <= 15 & downstream_dam_dist <= 15 ~ "cascade", 
    is.na(upstream_dam_dist) & is.na(downstream_dam_dist) ~ "FFR",
    is.na(upstream_dam_dist) | downstream_dam_dist <= 50 ~ "upstream", 
    upstream_dam_dist <= 50 | is.na(downstream_dam_dist) ~ "downstream", 
    TRUE ~ NA
  ))

# Column for casade only - not necessary 
dam_df <- dam_df %>% 
  mutate(cascade = ifelse(upstream_dam_dist <= 15 & downstream_dam_dist <= 15, 1, 0))

Need to make a pairwise comparisons of alternatives for each criteria for apply.APA

APA has 2 levels

  • Level 1 — Criteria weights (what we already did) “How important is connectivity vs CSI vs population vs PA?” → Produces a weight for each criterion

    • We have level 1 data because we already have numeric scores for each alternative on each criterion in the dataframe. Thius, we don’t need Level 2 AHP.

    • RMCDA’s AHP wants alternative comparisons too, but because we will have real scores we don’t need that — hence the dummy equal matrices just to extract the weights cleanly.

  • Level 2 — Alternative comparisons per criterion “For this specific criterion, how does Site A compare to Site B, Site C, etc.?” → Done separately for each criterion → Produces a score for each alternative on each criterion

    • Repeat this for every criterion — so with 4 criteria and 4 sites, that’s 4 separate matrices (why we make 4 matrices below)
# Create a Dummy matrices (for the apply.APA) that has the same col and row as dam_df
n_alternatives <- nrow(decision_matrix)
equal_matrix <- matrix(1, nrow = n_alternatives, 
                       ncol = n_alternatives,
                       dimnames = list(rownames(decision_matrix), 
                                       rownames(decision_matrix)))

comparing.competitors <- list(
  connectivity = equal_matrix,
  CSI          = equal_matrix,
  population   = equal_matrix,
  PA           = equal_matrix
)

Step 3: Use weights in TOPSIS

TOPIS reads the higher number a good

For population and pa, 0 = “good” and 1 = “bad”.

For CIS, lower = “good” and higher = “bad”.

Therefore, we need to flip those columns

decision_matrix_flipped <- decision_matrix

#Flip cost criteria (CSI, population, PA) so lower = better becomes higher = better
decision_matrix_flipped[, "CSI"]        <- 1 - decision_matrix[, "CSI"]
decision_matrix_flipped[, "population"] <- 1 - decision_matrix[, "population"]
decision_matrix_flipped[, "pa"]         <- 1 - decision_matrix[, "pa"]
# Giving the binary criteria more leverage in the normalization without changing their relative importance
